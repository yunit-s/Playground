# Flutter 기능 실험 프로젝트 종합 구현 계획

이 문서는 Flutter 기본 기능 실험을 위한 프로젝트의 개발 순서와 내용을 정의합니다.

## 단계 1: 핵심 구조 및 주요 기능 구현

### 1. 기본 앱 구조 설정 (하단 내비게이션 바)
- **목표:** 앱의 핵심 내비게이션 구조를 `BottomNavigationBar`로 구현합니다.
- **세부 작업:**
    1. 메인 스크린 위젯을 생성하여 `Scaffold`와 `BottomNavigationBar`를 포함시킵니다.
    2. 하단 바에 표시될 각 탭(예: 홈, 갤러리, 기본 기능)에 대한 빈 페이지 위젯을 생성합니다.
    3. 하단 바 탭을 누를 때마다 해당 페이지가 표시되도록 상태를 관리하고 화면을 전환하는 로직을 구현합니다.

### 2. 로컬 이미지 갤러리 기능 구현
- **목표:** 기기의 갤러리에서 사진을 선택하고 화면에 표시하는 기능을 구현합니다.
- **세부 작업:**
    1. '갤러리' 탭 페이지에 기능을 구현합니다.
    2. `image_picker` 패키지를 프로젝트에 추가합니다.
    3. '사진 추가' 버튼을 만들어 `image_picker`를 실행하고 사용자가 갤러리에서 여러 이미지를 선택할 수 있도록 합니다.
    4. 선택된 이미지들을 `GridView`를 사용하여 화면에 격자 형태로 표시합니다.

### 3. 기본 기능 페이지 구현
- **목표:** 다양한 Flutter의 기본 기능들을 테스트하고 모아볼 수 있는 페이지를 구현합니다.
- **세부 작업:**
    1. '기본 기능' 탭 페이지를 생성합니다.
    2. 기존의 Flutter 기본 카운터 앱을 이 페이지의 초기 콘텐츠로 포함시킵니다.
    3. **(하위 기능) 네트워크 통신 (API 호출):**
        - `http` 패키지를 프로젝트에 추가합니다.
        - [JSONPlaceholder](https://jsonplaceholder.typicode.com/)와 같은 무료 API를 사용하여 데이터를 요청합니다.
        - 응답받은 JSON 데이터를 Dart 객체로 변환(파싱)하고 `ListView`를 사용하여 화면에 표시하는 기능을 이 페이지 내에 구현합니다.

### 4. 메뉴 드로어 (Drawer) 구현
- **목표:** 보조적인 메뉴들을 담을 수 있는 사이드 메뉴(Drawer)를 추가합니다.
- **세부 작업:**
    1. 메인 `Scaffold`에 `drawer` 속성을 추가합니다.
    2. `Drawer` 내부에 '정보', '라이선스' 등 보조적인 페이지로 이동할 수 있는 메뉴 항목(`ListTile`)들을 배치합니다.

## 단계 2: 데이터 및 디바이스 연동 심화

### 5. 사용자 입력 폼 (Form) 페이지 구현
- **목표:** 설정이나 프로필 변경 등 사용자의 입력을 처리하는 폼 페이지를 구현합니다.
- **세부 작업:**
    1. '설정' 탭 또는 페이지를 새로 생성합니다.
    2. `Form`, `TextFormField` 위젯을 사용하여 입력 필드를 만들고, `ElevatedButton`으로 제출 기능을 구현합니다.
    3. 이메일 형식, 비밀번호 길이 등 입력값에 대한 유효성 검사(Validation) 로직을 추가합니다.

### 6. 간단한 로컬 데이터 저장 (Shared Preferences)
- **목표:** 앱 설정과 같은 간단한 데이터를 기기에 저장하여 앱을 재시작해도 유지되도록 합니다.
- **세부 작업:**
    1. `shared_preferences` 패키지를 추가합니다.
    2. '설정' 페이지에 다크 모드 전환 스위치 같은 UI를 추가합니다.
    3. 스위치 상태가 변경될 때마다 그 값을 `shared_preferences`에 저장하고, 앱 시작 시 저장된 값을 불러와 UI에 반영합니다.

### 7. 로컬 데이터베이스 (SQLite) 연동
- **목표:** 메모장이나 할 일 목록처럼 구조화된 데이터를 앱 내에 저장하고 관리하는 기능을 구현합니다.
- **세부 작업:**
    1. '메모' 탭 또는 페이지를 새로 생성합니다.
    2. `sqflite`와 `path` 패키지를 추가합니다.
    3. 데이터 모델 클래스(예: `Memo`)를 정의합니다.
    4. 데이터베이스 연결, 테이블 생성 및 CRUD(Create, Read, Update, Delete) 기능을 구현하여 메모를 관리합니다.

### 8. 위치 정보 (GPS) 사용
- **목표:** 기기의 현재 위치 정보를 가져와 화면에 표시합니다.
- **세부 작업:**
    1. '위치' 탭 또는 페이지를 새로 생성합니다.
    2. `geolocator` 패키지를 추가하고, 각 플랫폼(iOS/Android)에 맞는 위치 정보 사용 권한 설정을 추가합니다.
    3. 버튼을 누르면 현재 위도, 경도 값을 가져와 화면에 텍스트로 표시합니다.

### 9. 푸시 알림 (Firebase 연동)
- **목표:** Firebase Cloud Messaging(FCM)을 통해 원격 푸시 알림을 수신하는 기능을 구현합니다.
- **세부 작업:**
    1. Firebase 프로젝트 생성 및 Flutter 앱 연동 설정을 진행합니다.
    2. `firebase_core`, `firebase_messaging` 패키지를 추가합니다.
    3. 앱 시작 시 Firebase 초기화 및 알림 권한 요청 로직을 구현합니다.
    4. Firebase 콘솔을 통해 테스트 푸시 알림을 보내고 앱에서 수신되는 것을 확인합니다.

## 단계 3: 아키텍처 개선

### 10. 상태 관리(State Management) 도입
- **목표:** 앱의 규모가 커짐에 따라 복잡해지는 상태를 효율적으로 관리하기 위해 상태 관리 솔루션을 도입합니다.
- **세부 작업:**
    1. `Provider` 또는 `Riverpod`와 같은 상태 관리 패키지를 학습하고 프로젝트에 추가합니다.
    2. 앞서 구현한 기능 중 하나(예: 갤러리 또는 API 호출 기능)를 선택하여 `setState` 기반이 아닌 새로운 상태 관리 방식으로 리팩토링합니다.
